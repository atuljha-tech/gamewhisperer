## 🔥 The Problem

Sound design is one of the most overlooked and underestimated parts of game development. Most indie developers face the same painful cycle:

- Spend hours browsing stock sound libraries looking for something that fits
- Pay licensing fees for audio they might not even end up using
- Hire voice actors for NPC dialog and wait days for delivery
- Give up entirely and ship a silent game that feels incomplete

The average indie developer spends **2–3 days** on audio before a game jam deadline. That's time that should go into gameplay, mechanics, and polish.

**Game Whisperer makes that entire process disappear — in under 60 seconds.**

## ✨ How It Works

<img src="docs/screenshots/web-app-results.png" alt="AI Analysis Results" width="100%"/>

The workflow is dead simple:

**Step 1** — Take a screenshot of your game (any screenshot, any engine)

**Step 2** — Drop it into the Game Whisperer web app

**Step 3** — The Falcon AI vision model analyzes every visual element on screen — the terrain type, the character sprites, the objects, the environment style — and returns a complete structured sound design brief in under 10 seconds

**Step 4** — Each sound effect in the brief comes with:
- A unique `name` (e.g. `footstep_grass`)
- A `description` of exactly how it should sound (e.g. "soft crunching sound, like walking on dry grass")
- A `why` explaining its purpose in the game

**Step 5** — Copy the JSON. Paste it into Godot's `AiAnalyze` tool. Hit Import. Every sound effect is auto-configured as an `ElevenSFX` node. Hit Generate on each one — done.

## 🎮 The Demo Game

A fully playable Godot 4 top-down RPG where **every single sound was generated by AI**. Not recorded in a studio. Not purchased from a stock library. Generated by typing a plain English sentence into a text field.

<img src="docs/screenshots/gameplay-world.png" alt="Game World" width="100%"/>

The demo game is a sunny pixel-art world with trees, bushes, a house, NPCs, and coins — a classic RPG setup. Every audio event you experience while playing was produced by the ElevenLabs API in real time during development.

### What You Can Hear

| Feature | How it works |
|---|---|
| 👣 **Footsteps** | AI-generated grass crunch with random pitch variation (0.8x–1.2x) so it never sounds repetitive or robotic |
| 🪓 **Tree chopping** | Player switches to chopping animation when overlapping a tree — axe impact SFX fires on exactly frame 6 via `frame_changed` signal |
| 🪙 **Coin pickup** | Coins spawn, float upward 50px, and fade out — each plays an AI-generated coin jingle via `autoplay` |
| 🌿 **Bush rustle** | Walking into a bush area triggers an ambient disturb sound via `Area2D` overlap |
| 💬 **NPC gibberish** | First NPC plays AI-generated gibberish audio while typewriter text animates — duration calculated from character count |
| 🗣️ **AI voice dialog** | Second NPC uses ElevenLabs TTS — the typewriter animation duration is set to exactly match the audio length |
| 🎵 **Background music** | AI-generated ambient soundtrack loops throughout the entire game session |

## 🛠️ Godot Editor — AI Plugin Nodes

<img src="docs/screenshots/godot-editor.png" alt="Godot Editor" width="100%"/>

Four custom `@tool` nodes ship with this project. The `@tool` annotation means they run **inside the Godot editor itself** — not at runtime. This means the AI generation happens as part of your development workflow, not as a separate pipeline.

### ElevenSFX Node

The core node. Drop it anywhere in your scene tree, fill in two fields, click one button.

```
audio_name  →  "footstep"
description →  "soft grass footstep with slight crunch"
[Generate Audio] → MP3 saved to godotai/ instantly
```

Internally it calls `ElevenLabs /v1/sound-generation` with `duration_seconds: 0.5` and `prompt_influence: 0.8`, writes the binary response directly to the Godot filesystem, and stores the path in `do_not_edit_file_path` so it persists across editor restarts.

### ElevenDialog Node

Exactly like ElevenSFX but for voiced NPC dialog. Uses ElevenLabs `/v1/text-to-speech`.

```
voice_id  →  pick any ElevenLabs voice ID
dialog    →  "Welcome, traveller. Every sound you hear was made with AI."
[Generate Dialog Audio] → TTS MP3 synced to text duration
```

The `audio_length` getter returns the stream duration so other nodes can sync animations to it precisely.

### AiBgm Node

AI-generated background music via the Falcon AI Stable Audio model.

```
audio_name  →  "main_theme"
description →  "peaceful sunny day pixel art RPG theme, loopable, upbeat"
[Generate BGM] → WAV saved to godotai/bgm/
```

### AiAnalyze — One-Click Full Pipeline

The flagship feature. This node analyzes your entire game scene visually and scaffolds the complete audio layer automatically.

1. Select the `AiAnalyze` node in your scene
2. Click **"Analyze SFX with AI"** in the Inspector
3. Falcon AI vision model scans your scene screenshot
4. Returns a list of every sound effect your game needs with names and descriptions
5. Click **"Import"** — auto-creates a `_audio.tscn` packed scene with all `ElevenSFX` and `ElevenDialog` nodes pre-configured
6. Open that scene, hit Generate on each node — your entire game audio is done

## 🌐 Web App

<img src="docs/screenshots/gameplay-npc.png" alt="NPC Dialog" width="100%"/>

The companion Next.js web app provides a standalone interface for the AI sound design analysis — usable with any game, any engine, not just Godot.

**Tech stack:** Next.js 15, React 19, TypeScript, Tailwind CSS, react-dropzone

**How it works:**
- User drops a screenshot into the drag-and-drop zone
- Image is converted to base64 inline (no external file storage needed)
- Sent to `/api/upload` which calls the Falcon AI vision model
- Model returns structured JSON `{ fx: [{name, description, why}] }`
- Results render as interactive cards with name, description, and purpose
- Copy JSON button exports the full brief for use in Godot

## ⚡ Quick Start

### Prerequisites
- [Godot 4.4](https://godotengine.org/download)
- [Node.js 20+](https://nodejs.org)
- API keys from [ElevenLabs](https://elevenlabs.io) and [Falcon AI](https://falconllm.tii.ae)

### 1. Clone
```bash
git clone https://github.com/atuljha-tech/gamewhisperer.git
cd gamewhisperer
```

### 2. Set up API keys for Godot
```bash
echo "your_eleven_labs_key" > godot-game/ELEVEN_LABS_API_KEY.txt
echo "your_falcon_ai_key"   > godot-game/FAL_API_KEY.txt
```

> Both files are gitignored — they will never be committed.

**Where to get them:**
- **ElevenLabs** → [elevenlabs.io/app/settings/api-keys](https://elevenlabs.io/app/settings/api-keys) — enable **Sound Effects** + **Text to Speech** permissions
- **Falcon AI** → sign up at [falconllm.tii.ae](https://falconllm.tii.ae)

### 3. Set up the web app
```bash
cd web
cp .env.example .env.local
# paste your Falcon AI key into .env.local as GROQ_API_KEY
npm install
npm run dev
```
Open [http://localhost:3000](http://localhost:3000)

### 4. Open the game in Godot
1. Open Godot 4.4
2. Click **Import** → browse to `godot-game/project.godot` → Open
3. Wait for assets to import (~30 seconds on first launch)
4. Click **2D** in the top toolbar
5. Press **F5** to play
